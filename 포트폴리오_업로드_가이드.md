# 포트폴리오 업로드 가이드

새 프로젝트를 사이트에 추가하는 방법입니다.
수정 파일은 딱 하나, **`data/works.json`** 만 편집하면 됩니다.

---

## 1단계 — 이미지 준비

### 이미지 호스팅
현재 사이트의 모든 이미지는 **Webflow CDN**에 올라가 있습니다.
새 프로젝트를 올릴 때 이미지를 올릴 수 있는 방법은 두 가지입니다.

**방법 A — Webflow에 계속 올리기 (권장)**
1. Webflow 관리자 페이지 접속
2. Assets 탭에 이미지 업로드
3. 업로드된 이미지 URL 복사 (형식: `https://cdn.prod.website-files.com/...`)

**방법 B — 다른 이미지 호스팅 사용**
- [Cloudinary](https://cloudinary.com) — 무료 플랜 25GB
- [ImgBB](https://imgbb.com) — 무료
- AWS S3, Google Cloud Storage 등

어느 방법이든 **공개 접근 가능한 URL**이면 됩니다.

### 이미지 스펙
| 항목 | 권장 사양 |
|------|----------|
| 썸네일 | 1280×720px (16:9 비율) |
| 상세 이미지 | 1920 × 1080px |
| 형식 | PNG 또는 JPG |
| 파일 크기 | 썸네일 1MB 이하, 상세 이미지 2MB 이하 권장 |

## 2단계 — works.json 편집

`data/works.json` 파일을 열고, 배열의 **맨 앞** 또는 원하는 위치에 아래 형식으로 추가합니다.

### 필드 설명

```json
{
  "id": 10,
  "slug": "프로젝트-영문-이름",
  "title": "화면에 표시될 프로젝트 제목",
  "tags": "카테고리1, 카테고리2",
  "thumbnail": "https://썸네일-이미지-URL",
  "year": "2025",
  "duration": "Duration | 2025.01 - 2025.03",
  "description": "한국어 프로젝트 설명.\n줄바꿈은 \\n으로 표시합니다.",
  "descriptionEn": "English description of the project.\nLine breaks use \\n.",
  "images": [
    "https://상세-이미지-1-URL",
    "https://상세-이미지-2-URL",
    "https://상세-이미지-3-URL"
  ]
}
```

### 각 필드 작성 요령

| 필드 | 설명 | 예시 |
|------|------|------|
| `id` | 기존 마지막 id + 1로 순서대로 | `10` |
| `slug` | URL에 사용되는 고유 영문 이름. 소문자·하이픈만 사용, 중복 불가 | `"my-new-project"` |
| `title` | Works 그리드와 상세 페이지 상단에 표시 | `"My New Project"` |
| `tags` | 쉼표로 구분, Works 그리드 하단에 회색으로 표시 | `"Branding, Logo"` |
| `thumbnail` | Works 그리드에 표시되는 대표 이미지 URL | CDN URL |
| `year` | 작업 연도 (현재 상세 페이지에서 직접 표시되지는 않음) | `"2025"` |
| `duration` | 상세 페이지 사이드바에 표시 | `"Duration \| 2025.01 - 2025.03"` |
| `description` | 한국어 설명. `\n`으로 단락 구분 | — |
| `descriptionEn` | 영문 설명. 생략 가능 (없으면 영문 설명 영역이 숨겨짐) | — |
| `images` | 상세 페이지에 순서대로 표시되는 이미지 URL 배열 | — |

## 3단계 — 실제 추가 예시

`data/works.json` 파일을 열면 아래처럼 생겼습니다.

```json
[
  {
    "id": 1,
    "slug": "ancient-future-hair-dye",
    ...
  },
  ...
  {
    "id": 9,
    "slug": "30black",
    ...
  }
]
```

여기서 `]` 바로 앞에 `,`를 붙이고 새 항목을 추가합니다.

```json
[
  ...
  {
    "id": 9,
    "slug": "30black",
    ...
  },
  {
    "id": 10,
    "slug": "new-project",
    "title": "New Project",
    "tags": "Branding, Logo",
    "thumbnail": "https://이미지URL/thumbnail.png",
    "year": "2025",
    "duration": "Duration | 2025.01 - 2025.03",
    "description": "한국어 설명입니다.\n두 번째 줄입니다.",
    "descriptionEn": "English description.\nSecond line.",
    "images": [
      "https://이미지URL/01.png",
      "https://이미지URL/02.png"
    ]
  }
]
```

## 4단계 — 확인

저장 후 로컬 서버(`http://localhost:3000`)에서 바로 확인할 수 있습니다.

- **Works 그리드** → 메인 페이지에서 썸네일 확인
- **상세 페이지** → `http://localhost:3000/works/[slug]` 로 접속

---

## 자주 하는 실수

| 실수 | 증상 | 해결 |
|------|------|------|
| `slug` 중복 | 엉뚱한 페이지로 이동됨 | slug는 전체에서 유일하게 작성 |
| `slug`에 한글·공백 사용 | 페이지가 안 열림 | 영문 소문자와 하이픈(`-`)만 사용 |
| JSON 쉼표 누락/초과 | 사이트 전체가 오류남 | 마지막 항목 뒤에는 `,` 없어야 함 |
| 이미지 URL이 비공개 | 이미지가 안 뜸 | 브라우저에서 직접 URL 열어서 이미지 확인 |
| `\n`을 실제 줄바꿈으로 입력 | JSON 파싱 오류 | 줄바꿈은 반드시 `\n` 문자열로 표기 |

## 그리드 순서 변경

Works 그리드는 **works.json의 배열 순서대로** 표시됩니다.
위로 올리고 싶은 프로젝트를 배열 앞쪽으로 이동하면 됩니다.
(id 값은 바꾸지 않아도 됩니다.)
